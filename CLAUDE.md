# UAV Simulator 项目指令

## 设计原则

按照主流工业级做法进行方案设计与实施。

## 规范

- 注释用中文，日志用英文。
- 永远不要主动提交代码，除非我明确要求你这么做。

## Shell 环境

**Windows**: 所有命令通过 `cmd //c` 运行
**macOS/Linux**: 所有命令直接运行

## 工作流

每次修改 C++ 代码后，必须执行以下循环：

### Windows
1. **编译**: `cmd //c Script\\build.bat`
2. **编译失败** → 读取错误输出 → 修复代码 → 重新编译
3. **编译成功** → **运行仿真**: `cmd //c Script\\sim.bat`
4. **分析日志**: 读取 `Logs\uav.log`，从以下多个维度综合评估：
   - 与问题高度相关的日志
   - **速度**：巡航速度是否达标，避障时速度损失是否合理
   - **稳定性**：姿态/速度是否震荡，控制输出是否平滑
   - **避障轨迹合理性**：绕障路径是否流畅，有无不必要的大幅偏转或原地徘徊
   - `Warning` / `Error` 消息
5. **发现问题** → 修复代码，必要时可以添加重要日志 → 回到步骤 1
6. **行为正常** → 完成

### macOS/Linux
1. **编译**: `bash Script/build.sh`
2. **编译失败** → 读取错误输出 → 修复代码 → 重新编译
