## 1. 文档目标

本文定义无人机局部避障、求解器初始化、安全滤波和仿真优化工作流的统一设计。目标不是继续叠加补丁或孤立调参，而是建立以下闭环：

1. 使用可复现基准和结构化参数定位问题。
2. 保留 PGD，改进代价函数和初始化，解决对称场景中的局部最优。
3. 使用 CBF-QP 统一安全约束，并逐步替换散落的安全后处理。
4. 由 `.claude/skills/simulate/SKILL.md` 驱动构建、测试、仿真、度量和迭代。

本文只规划三个实施阶段。低维 CEM 是第二阶段中的条件功能，不单独形成阶段。

## 2. 当前实现判断

### 2.1 NMPC

当前 NMPC 具备以下基础：

- 双积分预测模型。
- 有限差分梯度。
- PGD、投影、回溯线搜索和 warm start。
- 球、Box、Cylinder 的近似有符号距离。
- 指数障碍代价、卡死检测和六方向扰动。

主要问题：

- 对称几何、对称初始化和对称标量代价共同造成绕行分支选择困难。
- 参考跟踪使用非平方欧氏距离，有限差分在零点附近不够平滑。
- 当前代价缺少显式路径进度和局部路径坐标信息。
- 六方向扰动对整个控制时域施加同一偏置，表达能力有限。
- 使用不同状态下的跨帧总代价判断 warm start 退化，比较对象并不一致。
- 当前速度投影会反推新的加速度，但不重新检查最大加速度，可能产生 `SatRatio > 1`。
- 卡死、最小控制注入、偏差保护和逃逸逻辑承担了部分求解器职责。

### 2.2 安全控制链

当前控制命令会依次经过偏差保护、速度钳位、CBF-QP、紧急制动、加速度限幅和紧急排斥。CBF 输出之后仍被修改，因此即使 CBF 约束正确，也不能保持约束成立。

现有机间 CBF 还存在以下问题：

- 二阶 CBF 中相对加速度符号错误。
- `MinHValue` 保存的是约束右端项，不是真实的 `h`。
- 逐约束投影没有 QP 最优性、不可行性和最终残差保证。
- 缺少松弛变量和执行器可行域。
- 紧急排斥仍作为 CBF 后置补丁存在。

因此，在安全层重写完成前，只能称当前逻辑为启发式碰撞缓解，不能称为条件安全保证。

## 4. 统一验收指标

### 4.1 硬门槛

以下指标不允许自动降级：

- 碰撞、障碍物穿透和机间碰撞次数为 0。
- 不出现 NaN、Inf、数组越界或求解器未处理失败。
- 不允许通过 `Overtime timeout`、`force completing` 等逻辑伪造任务成功。
- 正常飞行不出现持续姿态失稳；Roll/Pitch 超过 60 度视为失败。
- 安全层启用后，必须记录最终约束残差和 slack。
- CBF 输出后不得存在正常路径下的加速度修改器。

### 4.2 性能指标

默认值可由场景配置覆盖，但不能在一次优化任务中自动降低：

- 任务成功率：确定性场景 100%，随机回归场景不低于 95%。
- 巡航速度：稳定段不低于机型 `MaxVelocity` 的 50%。
- 避障后速度恢复：8 秒内恢复到 `MaxVelocity` 的 40%。
- 单次低速持续时间：低于 `MaxVelocity` 的 30% 不超过 15 秒。
- 横向轨迹偏差：默认不超过 5 米。
- 无连续 20 秒的负进度、原地徘徊或绕圈。
- NMPC 求解时间同时报告 P50、P95 和 P99，不能只报告平均值。

### 4.3 求解器指标

每次 NMPC 求解至少记录：

- `InitialCost`、`FinalCost`、`RelativeCostDecrease`。
- 投影梯度范数或等价的一阶最优性残差。
- 迭代次数、回溯次数、线搜索是否失败。
- 初值来源和最终选中的候选类型。
- 最小预测净空。
- 预测路径进度。
- 求解时间。

每次 CBF-QP 求解至少记录：

- 真实 `MinH`。
- 最大归一化约束残差。
- static slack 和 agent slack。
- 活跃约束数量。
- 求解状态和求解时间。

## 5. 阶段一：基线、参数和可复现性

### 5.1 目标

在修改求解器行为前，建立可信的实验基础，并隔离当前已知 CBF 风险。

### 5.2 参数结构

保留 `FNMPCConfig` 作为顶层配置，但按职责拆分嵌套结构：

```cpp
struct FNMPCSolverConfig;
struct FNMPCCostConfig;
struct FNMPCObstacleConfig;
struct FNMPCInitializationConfig;
struct FNMPCDiagnosticsConfig;

struct FSafetyFilterConfig;
struct FActuatorConstraintConfig;
struct FSimulationMetricConfig;
```

约束：

- 构造函数和函数体不再保存可调控制参数。
- 机型差异通过控制配置档案表达，不在 `AUAVPawn::BeginPlay` 中逐字段覆盖。
- 推荐使用 `UPrimaryDataAsset` 保存每种机型的控制档案。
- 物理常量、数值容差和调参参数分开管理。
- 只提取影响控制行为的魔法数字，不机械搬运所有数字。

参数关系必须在配置注释中明确：

- `PredictionHorizon / PredictionSteps = dt`。
- 障碍影响距离必须大于安全距离。
- 制动距离、最大速度和最大减速度必须一致。
- 有限差分步长应按控制尺度设置。
- 代价改为平方量后，旧权重不能直接复用。
- CBF 增益与控制周期、最大加速度和传感延迟相关。

### 5.3 基准场景

至少建立以下确定性回归：

| 场景 | 主要验证 |
|---|---|
| 无障碍直线 | 跟踪、速度、求解时间 |
| 单球正前方 | 单障碍对称绕行 |
| 左右对称双障碍 | 可通行走廊，不应错误绕开 |
| 对称封堵 | 必须选择左/右/上/下分支 |
| U 形障碍 | 局部最优和负进度 |
| 窄通道 | 净空和控制平滑性 |
| 动态横穿障碍 | 预测与制动 |
| 双机对向 | 机间 CBF 方向 |
| 三机汇聚 | 多约束可行性和 slack |

每个场景固定：

- 初始位置、速度和姿态。
- 障碍位置、尺寸和速度。
- 风场开关与随机种子。
- 飞行器型号。
- 仿真时长、`slomo` 和控制周期。

### 5.4 结构化日志

新增可机器解析的低频汇总日志，不依赖自然语言日志推断完整状态：

```text
[SIM_METRIC] Run=... Agent=... T=... Speed=... Progress=... CrossTrack=...
             Clearance=... Roll=... Pitch=... NMPCMs=... Stuck=...

[NMPC_SOLVE] Agent=... Init=Warm|Left|Right|Up|Down|Brake
             J0=... J=... Decrease=... Grad=... Clearance=...
             Progress=... Iter=... LineSearchFail=... Ms=...

[CBF_SOLVE] Agent=... MinH=... Residual=... StaticSlack=...
            AgentSlack=... Active=... Status=... Ms=...

[SIM_SUMMARY] Run=... Success=... ForcedComplete=... Collision=...
              MinClearance=... MaxTilt=... StuckCount=... ReplanCount=...
```

高频数据优先写 CSV；日志只保留节流后的诊断和最终摘要。

### 5.5 当前 CBF 风险隔离

完整重写前必须执行以下二选一：

1. 默认禁用当前 CBF-QP，只保留明确标记为启发式的旧安全链。
2. 立即修正机间约束符号，并补充“朝向邻机/远离邻机”的方向性单元测试。

这项工作属于已知缺陷控制，不等待第三阶段结束。

### 5.6 阶段验收

- 所有参数拥有单一来源。
- 至少 8 个确定性场景可重复运行。
- 相同版本重复运行三次，关键指标波动在约定范围内。
- 日志能够区分真实成功和强制完成。
- 当前 CBF 不再以错误符号运行。

## 6. 阶段二：PGD 鲁棒性优化

### 6.1 代价函数

#### 路径局部坐标

对每个参考点构造局部切向量：

\[
t_k = \frac{r_{k+1}-r_k}{\|r_{k+1}-r_k\|}
\]

位置误差分解为：

\[
e_k = p_k-r_k
\]

\[
e_{\parallel,k}=t_k^\top e_k,\qquad
e_{\perp,k}=e_k-t_k e_{\parallel,k}
\]

跟踪代价使用：

\[
J_{track} =
\sum_k
w_{lat}\|e_{\perp,k}\|^2 +
w_{lag}e_{\parallel,k}^2
\]

这样可以分别控制横向偏差和沿轨迹滞后，避免只使用欧氏距离。

#### 路径进度

不要使用无界负奖励。使用期望进度缺口：

\[
\Delta s =
\sum_k \max(0, t_k^\top(p_{k+1}-p_k))
\]

\[
J_{progress} =
w_{progress}\left[\max(0,\Delta s_{target}-\Delta s)\right]^2
\]

反向运动可增加单独惩罚：

\[
J_{reverse} =
w_{reverse}\sum_k
\left[\max(0,-t_k^\top(p_{k+1}-p_k))\right]^2
\]

#### 平方误差

参考位置和速度误差改为平方范数。修改后必须重新标定权重，因为量纲从厘米变为平方厘米，旧权重不再可比。

#### 障碍代价

继续使用已有有符号距离，不把“使用 SDF”当成消除对称性的手段。推荐平滑 hinge：

\[
\phi(d)=\frac{1}{\beta}\log(1+\exp(\beta(d_{safe}-d)))
\]

\[
J_{obs}=w_{obs}\sum_k \phi(d_k)^2
\]

在影响距离边界使用 `smoothstep` 衰减，避免硬截断造成梯度不连续。安全保证不由该软代价承担。

### 6.2 确定性多初值

删除求解后统一叠加的六方向扰动，改为求解前候选生成：

- `Warm`：上一帧控制序列左移。
- `Nominal`：沿参考轨迹加速。
- `Left`、`Right`：局部路径坐标中的水平横向绕行。
- `Up`、`Down`：垂向绕行。
- `Brake`：沿当前速度反向减速。

候选不直接使用 30 个独立变量生成。每个候选使用 2 至 4 个控制 knot，再插值得到完整控制序列。

计算预算：

1. 对所有候选执行前向 rollout 和约束投影。
2. 按代价、净空和进度淘汰明显劣质候选。
3. 对最优的 2 至 3 个候选执行短 PGD。
4. 对当前最优候选执行完整 PGD。

候选选择必须满足最小净空门槛。不能仅因代价低而选择预测碰撞轨迹。

### 6.3 绕行侧记忆

维护：

```cpp
enum class EAvoidanceHomotopy
{
    None,
    Left,
    Right,
    Above,
    Below
};
```

规则：

- 根据关键障碍附近预测轨迹在局部路径坐标中的平均偏移确定类别。
- 新类别只有在总代价改善超过切换迟滞，并满足净空要求时才能替换旧类别。
- 离开障碍影响区域、全局路径重规划或当前类别不可行时清除记忆。
- 记忆只形成软偏好，不能覆盖明显更安全的候选。

### 6.4 失败检测

只使用同一次优化问题内部的指标触发初始化器：

- 线搜索失败。
- 投影梯度残差小，但预测净空或进度不达标。
- `RelativeCostDecrease` 低于阈值。
- 预测路径连续负进度。
- warm start 候选被其他候选显著击败。

禁止使用相邻控制帧的原始 `TotalCost` 直接判断求解器退化。

### 6.5 控制序列投影

每个预测步需要同时满足：

\[
\|u_k\|\le a_{max}
\]

\[
\|v_k+\Delta t u_k\|\le v_{max}
\]

这是控制空间中两个球的交集。当前“先限制加速度，再限制速度并反推加速度”的顺序不能保证第一个约束仍成立。

实现要求：

- 使用 Dykstra 投影或等价的交集投影算法。
- 设置固定最大投影迭代数和可行性容差。
- 每次投影后同时检查加速度残差和速度残差。
- 若当前速度已超过上限，优先生成执行器可实现的最大制动控制，而不是反推出超出加速度能力的命令。
- 将最大投影残差写入 NMPC 诊断。

### 6.6 低维 CEM 准入

CEM 不默认实现。满足以下条件后才启用：

- 确定性多初值在固定回归集上的任务成功率仍低于 95%，或卡死率高于 5%。
- 失败样本确认属于初始化/非凸问题，而不是模型、代价尺度或安全层问题。
- NMPC P95 求解预算至少保留 30% 余量。

启用后：

- 只搜索 2 至 4 个 knot，不搜索完整控制序列。
- 初始均值来自最佳确定性候选。
- 默认 16 个样本、4 个 elite、最多 2 轮。
- 只在失败检测触发时运行，并设置冷却时间。
- CEM 只生成 PGD 初值，不直接输出最终控制。

### 6.7 阶段验收

- 对称封堵场景不依赖 `StuckEscapeCount` 或最小控制注入完成绕行。
- 走廊场景保持居中，不因人为破坏对称而无意义绕行。
- U 形和窄通道场景无长期负进度。
- 确定性场景任务成功率 100%。
- 卡死和全局重规划次数相对基线显著下降。
- 所有预测控制同时满足加速度和速度约束容差。
- P95 求解时间满足控制周期预算。
- 若确定性多初值已达标，不引入 CEM。

## 7. 阶段三：安全层重构

### 7.1 机间二阶 CBF

定义：

\[
\Delta p=p_i-p_j,\qquad
\Delta v=v_i-v_j
\]

\[
h=\|\Delta p\|^2-D_{safe}^2
\]

\[
\dot h=2\Delta p^\top\Delta v
\]

\[
\ddot h=
2\|\Delta v\|^2+
2\Delta p^\top(u_i-u_j)
\]

采用：

\[
\ddot h+\alpha_1\dot h+\alpha_0h\ge-\xi_{agent}
\]

转换为 `A u <= b`：

\[
-2\Delta p^\top u_i-\xi_{agent}
\le
2\|\Delta v\|^2
-2\Delta p^\top u_j
+\alpha_1\dot h
+\alpha_0h
\]

实现和测试必须以该公式为唯一来源，避免注释与代码分别维护。

### 7.2 静态障碍 HOCBF

对静态障碍使用：

\[
h=d(p)-d_{safe}
\]

\[
\dot h=\nabla d^\top v
\]

\[
\ddot h=\nabla d^\top u+v^\top H_dv
\]

约束：

\[
\ddot h+\alpha_1\dot h+\alpha_0h\ge-\xi_{static}
\]

球体可以使用解析梯度和 Hessian。Box/Cylinder 在棱边处不光滑，第一版使用最近点法线和保守局部半空间，并在切换法线时进行迟滞；后续再评估平滑 SDF。

动态障碍应使用相对位置、相对速度和可用的障碍加速度估计。

### 7.3 QP

决策变量：

\[
z=[u_x,u_y,u_z,\xi_{static},\xi_{agent}]
\]

目标：

\[
\min_z
\frac12(u-u_{nom})^\top W(u-u_{nom})
+\rho_s\xi_{static}^2
+\rho_a\xi_{agent}^2
\]

约束包括：

- 静态障碍 CBF。
- 动态障碍 CBF。
- 机间 CBF。
- `xi_static >= 0`、`xi_agent >= 0`。
- 加速度可行域。
- 水平加速度/倾角可行域。
- 垂向推力可行域。
- 必要时加入下一控制周期的速度边界。

二范数边界使用保守多面体近似，保持问题为线性约束 QP。

### 7.4 求解器要求

不能继续使用一次次投影作为 QP 求解器。使用经过测试的小规模稠密 active-set 求解器，或引入可接受的成熟 QP 库。

无论采用哪种实现，都必须提供：

- 明确的 `Solved`、`SolvedWithSlack`、`MaxIterations`、`NumericalFailure` 状态。
- KKT 或等价最优性残差。
- 最大约束违反量。
- warm start。
- 固定最大迭代次数和确定性执行时间。

由于 slack 无上界且零加速度在执行器可行域内，QP 应保持数值可行。slack 大于零表示条件安全约束无法在当前执行器能力内严格满足，不表示求解成功即安全。

### 7.5 控制链迁移

按 shadow mode 迁移，不一次删除全部旧逻辑：

1. 新 CBF-QP 只计算和记录，不修改控制。
2. 验证约束方向、残差、slack 和计算时间。
3. 启用静态障碍约束，保留旧补丁作为监控但不重复修改命令。
4. 启用机间约束。
5. 将速度、横向加速度和最大加速度限制移入 QP。
6. 逐项删除紧急排斥、速度钳位和正常路径紧急制动。

故障监督器规则：

- 正常情况下，将紧急制动意图作为 `u_nom` 输入安全 QP。
- QP 数值失败时进入明确的 degraded mode，执行保守制动并触发重规划。
- degraded mode 不宣称 CBF 保证，必须记录故障原因。
- 硬电机停机只用于碰撞、动力系统故障或不可恢复状态。

### 7.6 阶段验收

- CBF 方向性、边界、逼近速度和多约束测试全部通过。
- shadow mode 中最大归一化残差满足容差。
- 正常模式中 CBF 后无加速度修改器。
- 双机对向和三机汇聚场景零碰撞。
- slack 使用率和最大 slack 在验收范围内。
- 删除安全补丁后，安全指标不低于旧基线，性能指标改善或持平。

## 8. 测试策略

### 8.1 单元测试

- 距离、梯度和局部路径坐标。
- 平滑 hinge 的值和有限差分梯度。
- 多初值生成及投影。
- homotopy 分类、迟滞和清除。
- CBF `h`、`hdot` 和约束方向。
- QP 单约束、多约束、带 slack 和执行器不可满足场景。

### 8.2 集成测试

- 对每个确定性场景运行固定时长。
- 输出 `[SIM_SUMMARY]`。
- 将强制完成视为失败。
- 对 solver P50/P95/P99、最小净空和最大姿态进行断言。

### 8.3 回归原则

- 每个实验使用同一场景、种子和运行参数。
- 同时保存基线和候选结果。
- 一次只验证一个明确假设。
- 安全失败直接拒绝候选，不用性能提升抵消。
- 不允许为了让当前候选通过而自动降低验收线。

## 9. `simulate` Skill 的职责

`.claude/skills/simulate/SKILL.md` 是本文的执行入口，负责：

- 选择本次关注范围。
- 检查工作区和当前基线。
- 构建、测试并运行可复现实验。
- 提取硬门槛、性能和求解器指标。
- 形成单一根因假设。
- 实施小范围修改。
- 使用同一场景回归。
- 接受有效修改或仅撤销本轮自身修改。
- 在达到迭代预算时给出证据和剩余风险，而不是降低目标。

Skill 不负责：

- 自动放宽安全或性能指标。
- 同时大改 NMPC、CBF 和姿态控制器。
- 把日志中没有出现错误当成安全证据。
- 将任务强制完成当成成功。
- 未经实验准入直接引入 CEM。
