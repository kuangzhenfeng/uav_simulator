# Simulate Skill

## 概述

运行仿真并自主修复所有问题，直到无人机能稳定快速飞行且轨迹跟踪达标。

## 判定标准

| 指标 | 达标 | 不达标 |
|------|------|--------|
| 巡航速度 | ≥ 8 m/s（维持稳定） | < 5 m/s 持续超过 10s |
| 停止时间 | 单次 ≤ 3s（如避障急停后立即恢复） | 单次 > 5s 或累计占比 > 10% |
| 低速持续时间 | 单次 ≤ 10s | 单次 > 20s 或频繁反复出现 |
| 速度恢复 | 降速后 5s 内恢复到 ≥ 6 m/s | 降速后迟迟不恢复 |
| 整体进度 | 仿真期间有明显位移，持续向目标推进 | 长时间原地徘徊或绕圈 |
| 姿态稳定性 | 无异常震荡，控制输出平滑 | 姿态/速度剧烈震荡 |
| 避障轨迹 | 绕障路径流畅，无大幅偏转 | 不必要的偏转或原地徘徊 |
| 轨迹跟踪偏差 | 所有 UAV 偏差 ≤ 3m | 存在 UAV 偏差 > 3m |

> 以上阈值为默认参考值，可根据实际场景调整。如果用户指定了具体目标速度或航线，以用户要求为准。

## 执行流程

### 1. 并行准备

同时进行以下操作：

- **后台运行仿真**（`run_in_background: true`）：
  - Windows: `Script\sim.bat 80`
  - macOS/Linux: `Script/sim.sh 80`
- **读取核心代码**，重点关注与速度/运动/避障相关的逻辑：
  - `Source\uav_simulator\Core\UAVPawn.cpp/h` — 飞行动力学与速度控制
  - `Source\uav_simulator\Planning\PathPlanner.cpp/h` — 路径规划与速度指令
  - `Source\uav_simulator\Planning\NMPCAvoidance.cpp/h` — 避障控制与速度约束
  - `Source\uav_simulator\Planning\ObstacleManager.cpp/h` — 障碍物管理
  - 其他与飞行控制、速度限制相关的文件

### 2. 读取日志并分析

等待仿真完成后读取 `Logs/uav.log`，重点提取：

- **速度数据**：各 UAV 的实时速度、平均速度、最低速度
- **停止事件**：速度接近 0 的时间点和持续时长
- **低速事件**：速度 < 5 m/s 的时间点和持续时长
- **速度恢复**：降速后恢复到巡航速度的耗时
- **位置变化**：是否有持续位移，还是原地不动
- **姿态数据**：是否存在异常震荡
- **轨迹偏差**：各 UAV 偏差是否 ≤ 3m

### 3. 对照判定标准评估

逐条检查判定标准表中的 8 项指标，标记每项为"达标"或"不达标"。

### 4. 若不达标 → 定位根因并修复

根据不达标项，定位根本原因：

| 不达标现象 | 常见根因 |
|-----------|---------|
| 长时间停止 | 避障过于保守、安全距离过大、路径被完全阻断 |
| 长时间低速 | 速度上限被限制、避障降速后不恢复、控制器增益不足 |
| 速度不恢复 | 缺少速度恢复逻辑、目标速度设置过低、NMPC 约束过紧 |
| 原地徘徊 | 路径规划死锁、目标点不可达、避障方向选择错误 |
| 姿态震荡 | 控制器增益过大、阻尼不足、反馈信号噪声 |
| 避障轨迹不流畅 | 避障算法参数不当、障碍物检测精度不够 |
| 轨迹偏差过大 | 路径跟踪算法问题、控制输出饱和、目标更新延迟 |

- 读取相关源码，定位具体代码位置
- 实施修复
- 编译：Windows `Script\build.bat` / macOS `Script/build.sh`
- 编译失败 → 修复编译错误 → 重新编译
- 编译成功 → 回到步骤 1

### 5. 每次迭代记录

记录本轮：
- 修改了什么文件、哪些代码
- 为什么修改（对应哪个不达标项）
- 修改前后的关键数据对比

### 6. 全部达标 → 输出总结

当所有指标达标时，输出：
- 各项指标的最终数据
- 总迭代次数和修改摘要
- 确认飞行稳定快速、轨迹跟踪达标

## 约束

- 不要在迭代之间询问确认，保持自主运行
