# 简化日志

**核心原则：日志应精炼、有节制、等级准确——宁可少打一条也不刷屏淹没关键信息。**

## 规则

1. **禁止 Verbose 等级**：不使用 `VeryVerbose` 和 `Verbose`，除非用户明确要求。
2. **高频日志必须节流**：在 Tick、每帧回调、循环体内等高频路径，必须使用 `UE_LOG_THROTTLE` 或 `UE_LOG_EVERY_N` 限制频率，或者直接删除不必要的日志。
3. **等级准确**：`Warning` 仅用于异常/错误情况，常规流程信息使用 `Log`。装饰性分隔线（如 `========`）不应使用 `Warning`。

## 执行流程

### 1. 扫描目标文件

用 `Grep` 搜索 `UE_LOG` 调用，识别：
- 未节流的日志（尤其在 Tick / 循环 / 回调中）
- 滥用 `Warning` 等级的常规日志
- `Verbose` / `VeryVerbose` 等级的使用

### 2. 逐条评估

| 场景 | 处理方式 |
|------|---------|
| Tick/每帧中的普通 `UE_LOG` | 改为 `UE_LOG_THROTTLE`（时间节流）或 `UE_LOG_EVERY_N`（次数节流），或删除 |
| 循环体内的 `UE_LOG` | 合并为循环外的一条汇总日志，或加节流 |
| `Warning` 等级的常规流程信息 | 降为 `Log` |
| `Verbose` / `VeryVerbose` 日志 | 删除或改为 `Log`（除非用户明确要求） |
| 装饰性分隔线日志 | 删除，或降为 `Log` |
| 仅打印中间变量且不影响调试 | 直接删除 |

### 3. 选择节流方式

| 条件 | 推荐方式 | 示例 |
|------|---------|------|
| 按固定时间间隔输出 | `UE_LOG_THROTTLE(Seconds, ...)` | `UE_LOG_THROTTLE(1.0, LogUAVActor, Log, TEXT("status: %s"), ...)` |
| 按调用次数输出 | `UE_LOG_EVERY_N(Count, ...)` | `UE_LOG_EVERY_N(10, LogUAVPosition, Log, TEXT("error: %.2f"), ...)` |
| 日志无实际调试价值 | 直接删除 | 中间变量仅用于日志 |

### 4. 编译验证

每次修改后执行编译，确保无编译错误。

## 常见误区

| 误区 | 正确做法 |
|------|---------|
| 用 `Verbose` 等级代替节流 | 禁止 Verbose；用节流宏控制频率 |
| 保留装饰性分隔线但改为 `Log` | 直接删除，它们不提供结构化信息 |
| 给所有日志加节流 | 仅对高频路径（Tick/循环/回调）加节流，事件驱动的日志无需节流 |
| 节流间隔过短（如 0.01s） | 根据实际场景选择合理间隔（通常 ≥ 0.5s） |
